# System Design LLD Learning Repo

This repository is organized as a learning path for Low Level Design (LLD). Start from the basics and move forward in the order below.

## Ordered Learning Path

1. [01_Intro to CPP](01_Intro%20to%20CPP/) - Start here for C++ basics, syntax, and simple problem solving.
2. [02_OOP Fundamentals](02_OOP%20Fundamentals/) - Learn encapsulation, inheritance, polymorphism, and abstraction.
3. [03_SOLID Principles](03_SOLID%20Principles/) - Understand how to design cleaner and more maintainable classes.
4. [04_Design Patterns](04_Design%20Patterns/) - Study Builder, Singleton, and creational design pattern examples.
5. [05_Functors](05_Functors/) - Practice comparator logic and function-object based sorting examples.
6. [06_Intro to Java](06_Intro%20to%20Java/) - Build Java fundamentals and beginner project structure.
7. [07_LLD Case Studies](07_LLD%20Case%20Studies/) - Explore larger LLD case studies and practice projects.

## Recommended Study Order Inside `07_LLD Case Studies`

1. [Car Parking Lot System](07_LLD%20Case%20Studies/Car%20Parking%20Lot%20System/) - Model parking spots, tickets, vehicles, and payments.
2. [Meeting System](07_LLD%20Case%20Studies/Meeting%20System/) - Practice scheduling, intervals, rooms, and notifications.
3. [Car Rental System](07_LLD%20Case%20Studies/Car%20Rental%20System/) - Study branches, booking flow, payments, and decorators.

## Notes

- Generated folders such as `output/`, `bin/`, `build/`, and `target/` are ignored from Git.
- Keep source files, diagrams, and notes under the topic folder they belong to.
- If you add a new LLD topic, place it in the sequence that best matches the learning path above.

## Suggested Flow

```text
Basics -> OOP -> SOLID -> Design Patterns -> Java Basics / C++ Basics -> LLD Projects
```

That flow will help you move from language fundamentals to real object-oriented design problems without jumping around.
